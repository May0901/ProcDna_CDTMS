Clinical Trial Data Management

Important:
    Features I have covered:
        1. RESTful APIs to handle CRUD operations for clinical trial data.
        2. GraphQL endpoint to fetch clinical trails and associated participants
        3. Used mongoDb for user/auth
        4. PostgreSQL for clinical trial and participants data.
        5. Basic Authentication and Authorization
        6. Express.js for routing and middleware
        7. Search and Filtering capabilty
        8. Dockerize the application

    Features I have not covered due to time constraints:
        1. Graphs (d3.js)
        2. Input validation and sanitization
        3. Server side rendering
        4. Updating participants (to save time as i already implemented the same for clinical trials)
        5. Notification System (dialog/alerts UI for notifying user of issues/errors)


Technology Used:
    Frontend:
        1. React
        2. MUI as a component library
        3. Zustand for state Management
        4. React router for routing

    Backend:
        1. NodeJs
        2. Express.js as a http server framework
        3. Sequelize as a ORM for postgreSQL
        4. Mongoose as a ODM for mongoDb
        5. brcypt.js for hashing password
        6. cors for Cross origin resource sharing
        7. jsonwebtoken for token based Authentication
        8. graphql lib for graphql endpoint



API Endpoints:
    Authentication:
        1. POST api/auth/signin  -> For Authentication
        2. POST api/auth/signup  -> For creating new user

    Clinical trial:
        1. GraphQL:Post api/trial/gql -> fetch all clinical trials and associated participants
        2. POST api/trial -> create new clinical trial
        3. DELETE api/trial/:id -> delete clinical trial by Id
        4. PATCH api/trial/:id -> update clinical trial by id

    Participants: 
        1. GET api/trial/:id/participant -> fetch all participants (not used)
        2. POST api/trail/:id/participant -> create new participant for a clinical trial
        3. DELETE api/trial/:trialId/participant/:participantId -> delete a participant by id associated to clinical trial


Data Models:
    User:
        _id: ObjecyId (autogenerated)
        name: string
        email: string
        password: string
        role: string (either admin or viewer)

    Clinical Trial:
        id: Integer (PK)
        title: string
        objective: string
        status: string
        phase: string
        location: string
        start_date: dateOnly
        end_date: dateOnly
        therapeutic_area: string

    Participant:
        id: Integer (PK)
        name: string
        gender: string
        age: string
        location: string
        enrollment_status: string (not used)
